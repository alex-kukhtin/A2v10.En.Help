Прикладные приложения

Система A2v10 предоставляет разработчику высокоуровневый предметно-ориентированный «конструктор» прикладных приложений.

Фактически прикладное приложение представляет собой простой каталог с файлами. Это описания моделей, представления, шаблоны, отчеты, статические файлы и т.д. Отметим, что прикладное приложение является «динамическим» в том смысле, что после изменений в файлах приложения, не требуется компиляции файлов, сборки проекта и его развертывания.

Каждый конечный каталог предствляет собой конечную точку (endpoint), которая адресуется через уникальный url-адрес.

Конечная точка (endpoint) представляет собой точку взаимодействия с приложением. Как правило, одна конечная точка отвечает за обработку одного элемента предметной области. Обращение к конечной точке выполняется по относительному пути (указывается в url). В каталоге, представляющем конечную точку, обязательно должен присутствовать файл model.json, в котором описаны возможные действия с этой конечной точкой. Можно считать, что признаком конечной точки является наличие в соответствующем каталоге файла model.json.

Система поддерживает переадресацию url-адресов в различные конечные точки. Для этого используется файл перенаправлений redirect.json, который должен находиться в корневом каталоге приложения.

Папка приложения может содержать дополнительные специальные папки.
